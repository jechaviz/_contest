# Rules Checklist

Source snapshot: 2026-05-29, verified against Devpost rules and overview.

Contest: Build with Gemini XPRIZE.

Deadline: 2026-08-17 1:00 PM PDT.

Primary submission category: Entrepreneurship & Job Creation.

## Eligibility

- [ ] Entrant is an individual above legal age of majority, an eligible team, or
  an organization with fewer than 25 employees.
- [ ] If submitting as a team or organization, one representative is authorized
  to submit.
- [ ] Entrant is not resident/domiciled in an excluded jurisdiction.
- [ ] Entrant has no conflict with XPRIZE, Devpost, judges, promotion entities,
  or their affiliates.
- [ ] If organization submits, corporate ID is available for the form.

## Project Requirements

- [ ] Project was newly created after 2026-05-19.
- [ ] Any pre-existing framework, boilerplate, local library, or template is
  disclosed with a clear explanation of what is new.
- [ ] Product is a business that operates with AI.
- [ ] Demo shows AI transforming core workflows, not only a chat add-on.
- [ ] At least one Google Cloud product is used in the running product.
- [ ] At least one deployed LLM call uses the Gemini API.
- [ ] All third-party APIs, SDKs, datasets, and UI assets are authorized.
- [ ] Product runs on the platform listed in the submission.

## Submission Requirements

- [ ] Category selected in Devpost.
- [ ] Code repository URL included.
- [ ] Repository is public or private-shared with `testing@devpost.com` and
  `judging@hacker.fund`.
- [ ] Repository contains all required source code, setup instructions, test
  instructions, and deployment notes.
- [ ] Text description explains category relevance and how requirements are met.
- [ ] Demo video is public on YouTube, Vimeo, or Youku.
- [ ] Video is under 3 minutes and shows the product functioning.
- [ ] Video avoids unauthorized trademarks, copyrighted music, or protected
  third-party material.
- [ ] If materials are not in English, English translation is provided.
- [ ] Judges receive free access without restrictions through the judging period.
- [ ] Private demo credentials, if any, are included only in Devpost testing
  instructions or a secure judge-only channel.

## Revenue Evidence

- [ ] Total arms-length third-party revenue during 2026-05-19 to 2026-08-17.
- [ ] Revenue by month: May, June, July, August 2026.
- [ ] Related-party revenue reported separately.
- [ ] Total costs during hackathon period excluding marketing acquisition spend.
- [ ] One-sentence cost description.
- [ ] Marketing and customer acquisition spend reported, even if zero.
- [ ] Invoices, receipts, payment processor exports, or bank statements are
  stored in the private evidence ledger.

## User Evidence

- [ ] Number of individual users.
- [ ] High-level user breakdown.
- [ ] Testimonials or feedback with user awareness/consent.
- [ ] Optional customer contact list ready for verification requests.
- [ ] Live demo path can be shown in a call within 2 business days.

## Running Product Evidence

- [ ] Agent execution logs.
- [ ] API usage records, including Gemini and Google Cloud.
- [ ] Screenshots of dashboards.
- [ ] Production playbook receipts.
- [ ] Monitoring or uptime evidence.
- [ ] Evidence that key decisions are executed by AI in production.

## Judging Alignment

- [ ] Business Viability: real launch, users, revenue, sustainable model.
- [ ] AI-Native Operations: AI live in production, executing decisions.
- [ ] Category Impact: credible improvement for founders and small teams.
